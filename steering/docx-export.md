# SecOps Report — DOCX Export Rules

Khi user yêu cầu xuất report sang `.docx`, agent PHẢI follow quy trình và style guide dưới đây.

---

## Khi nào áp dụng

- User nói: "xuất docx", "export word", "tạo file word", "gửi report"
- User muốn gom nhiều files thành 1 document hoàn chỉnh
- User cần share report với stakeholders (email, audit, management)

---

## Scope — Nội dung gom vào 1 file .docx

Gom TẤT CẢ per-scan artifacts + state thành 1 document có cấu trúc:

```markdown
# IaC Security Report — {project_name}

## 1. Executive Summary
← Lấy từ: summary.md (Overview + By Severity tables)

## 2. Scan Context
← Lấy từ: metadata.md (full table)

## 3. Findings Detail
← Lấy từ: remediation-plan.md (Priority Matrix + ALL priority sections)

## 4. Tech Debt Register
← Lấy từ: tech-debt.md (Accepted Debt + Needs Review + Suppressions)

## 5. Delta (nếu scan #002+)
← Lấy từ: delta.md (Summary + New/Resolved tables)

## 6. Scan History
← Lấy từ: state/tracking.md (Timeline table)

## 7. Project Context
← Lấy từ: state/project-memory.md (Context + Known Patterns + Decisions)

## Appendix A: Scan Plan
← Lấy từ: plan.md

## Appendix B: Severity Classification
← Lấy từ: remediation-plan.md (Severity Classification Rules table)
```

### Source files → Document sections mapping

| Section | Source file | Ghi chú |
|---------|-----------|---------|
| Executive Summary | `scans/{NNN}/summary.md` | Overview + Severity + Recommended Actions |
| Scan Context | `scans/{NNN}/metadata.md` | Full table |
| Findings Detail | `scans/{NNN}/remediation-plan.md` | ALL sections (P0→P3) |
| Tech Debt | `scans/{NNN}/tech-debt.md` | Full content |
| Delta | `scans/{NNN}/delta.md` | Chỉ khi scan #002+ |
| Scan History | `state/tracking.md` | Timeline + Remediation Log |
| Project Context | `state/project-memory.md` | Context + Patterns + Decisions |
| Scan Plan | `scans/{NNN}/plan.md` | Appendix |
| Classification Rules | `remediation-plan.md` template | Appendix — severity lookup |

## Workflow — 3 bước

### Step 1: Assemble Markdown

Agent gom nội dung từ source files thành 1 file `.md` theo cấu trúc trên.

**Rules:**
- Giữ nguyên tables (không reformat)
- Thêm page break markers `\newpage` trước mỗi H1 section
- Thêm header metadata block:

```markdown
---
title: "IaC Security Report — {project_name}"
subtitle: "Scan #{NNN} — {YYYY-MM-DD}"
author: "Generated by IaC Checkov AWS Power"
date: "{YYYY-MM-DD}"
---
```

- Loại bỏ agent-only sections ("Agent Rules", "Template version" footers)
- Loại bỏ internal paths (absolute paths → relative hoặc redact)

### Step 2: Convert to .docx (Pandoc)

```bash
pandoc {assembled}.md \
  -o "{project_name}-security-report-{NNN}.docx" \
  --toc --toc-depth=3 \
  --wrap=none \
  --columns=10000
```

### Step 3: Post-process (python-docx)

Chạy post-processing script để fix fonts, colors, table styling.

**Output file**: `.checkov-reports/reports/{project_name}-security-report-{NNN}.docx`

## Post-Processing Script

```python
from docx import Document
from docx.shared import Pt, Cm, RGBColor
from docx.oxml.ns import qn, nsdecls
from docx.oxml import parse_xml

doc = Document('output.docx')

def set_run_font(run, font_name, font_size=None):
    """Set font on run including rFonts XML."""
    run.font.name = font_name
    if font_size:
        run.font.size = font_size
    rPr = run._r.get_or_add_rPr()
    rFonts = rPr.find(qn('w:rFonts'))
    if rFonts is None:
        rFonts = parse_xml(
            f'<w:rFonts {nsdecls("w")} '
            f'w:ascii="{font_name}" w:hAnsi="{font_name}" '
            f'w:eastAsia="{font_name}" w:cs="{font_name}"/>'
        )
        rPr.insert(0, rFonts)
    else:
        for attr in ('w:ascii', 'w:hAnsi', 'w:eastAsia', 'w:cs'):
            rFonts.set(qn(attr), font_name)

# --- Page Setup ---
for section in doc.sections:
    section.page_width = Cm(21.0)
    section.page_height = Cm(29.7)
    section.left_margin = Cm(2.0)
    section.right_margin = Cm(1.7)
    section.top_margin = Cm(2.0)
    section.bottom_margin = Cm(2.2)

# --- Heading configs ---
heading_configs = {
    'Heading 1': {'size': Pt(16), 'bold': True, 'color': RGBColor(0, 0, 0)},
    'Heading 2': {'size': Pt(13), 'bold': True, 'color': RGBColor(0, 0, 0)},
    'Heading 3': {'size': Pt(12), 'bold': True, 'color': RGBColor(0x3E, 0x00, 0xFF)},
    'Heading 4': {'size': Pt(11), 'bold': True, 'color': RGBColor(0x3E, 0x00, 0xFF)},
}

# --- Process paragraphs ---
for para in doc.paragraphs:
    style_name = para.style.name
    if style_name.startswith('Heading'):
        cfg = heading_configs.get(style_name)
        for run in para.runs:
            set_run_font(run, 'Arial', cfg['size'] if cfg else Pt(12))
            if cfg:
                run.font.bold = cfg['bold']
                run.font.color.rgb = cfg['color']
    else:
        for run in para.runs:
            if not run._r.findall(qn('w:drawing')):
                set_run_font(run, 'Times New Roman', Pt(12))

# --- Fix tables ---
for table in doc.tables:
    for i, row in enumerate(table.rows):
        for cell in row.cells:
            for para in cell.paragraphs:
                for run in para.runs:
                    set_run_font(run, 'Arial', Pt(10))
                if i == 0:
                    for run in para.runs:
                        run.bold = True
            if i == 0:
                tcPr = cell._tc.get_or_add_tcPr()
                existing = tcPr.find(qn('w:shd'))
                if existing is not None:
                    tcPr.remove(existing)
                tcPr.append(parse_xml(
                    f'<w:shd {nsdecls("w")} w:fill="F2F2F2" w:val="clear"/>'
                ))

# --- Page breaks before H1 ---
first_h1 = True
for para in doc.paragraphs:
    if para.style.name == 'Heading 1':
        if first_h1:
            first_h1 = False
            continue
        pPr = para._p.get_or_add_pPr()
        existing = pPr.find(qn('w:pageBreakBefore'))
        if existing is not None:
            pPr.remove(existing)
        pPr.append(parse_xml(
            f'<w:pageBreakBefore {nsdecls("w")} w:val="true"/>'
        ))

doc.save('output.docx')
```

## Markdown Source Rules (trước khi export)

### Blank line TRƯỚC mọi block element

```markdown
<!-- ĐÚNG -->
**Business Rules:**

- BR1: Rule one.
- BR2: Rule two.

<!-- SAI — bullets inline với paragraph trước -->
**Business Rules:**
- BR1: Rule one.
```

### Tables KHÔNG có blank line giữa rows

```markdown
<!-- ĐÚNG — liên tục -->
| A | B |
|---|---|
| 1 | 2 |
| 3 | 4 |

<!-- SAI — pandoc tách thành 2 tables -->
| A | B |
|---|---|
| 1 | 2 |

| 3 | 4 |
```

### Metadata fields cách nhau 1 blank line

```markdown
<!-- ĐÚNG — mỗi field là paragraph riêng trong docx -->
**Date:** 2026-05-28

**Project:** acb-terraform-custom

**Scan #:** 001
```

## Anti-Patterns (KHÔNG BAO GIỜ)

| DO NOT | Lý do |
|--------|-------|
| Inject `numPr` XML vào bullet items | Gây "số nhảy lung tung" — pandoc quản lý numbering |
| Đổi style name của list items | Break pandoc's list structure → bullets biến mất |
| Dùng `--reference-doc` pandoc flag | Dễ conflict style — chỉ dùng post-processing |
| Để emoji trong output (✅❌🔴) | Render sai trong .docx — dùng text markers |
| Include "Agent Rules" sections | Đây là internal instructions, không cho stakeholders |
| Include absolute file paths | Security risk — dùng relative paths hoặc redact |
